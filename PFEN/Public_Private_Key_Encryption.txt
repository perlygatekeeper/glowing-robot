The magic: It's trivially easy to multiply p × q together to get n, but it's

Anyone can encrypt a message using your public key (which includes n)
Only you can decrypt it using your private key (which is derived from the secret p and q)

Even if an attacker intercepts the encrypted message and knows your public key,
they'd have to factor n to break it — which for large enough primes would take
longer than the age of the universe with current computers

the actual math used when n is used to encrypt the message?6:31 PMHere's the
actual math:

Setup:

Choose primes p and q, compute n = p × q
Compute Euler's totient: φ(n) = (p-1)(q-1)
Choose a public exponent e (commonly 65537)
Compute private exponent d such that e × d ≡ 1 (mod φ(n)) — meaning d is the modular inverse of e

Public key = (n, e)
Private key = (n, d)

Encryption:
Convert your message to a number m, then:
c = m^e mod n
c is the ciphertext sent over the wire.

Decryption:
m = c^d mod n

That's it! Raise the ciphertext to the power of d, mod n, and you get the
original message back.

Why it works is based on Fermat's little theorem — the math guarantees that
(m^e)^d mod n = m, but only if you know d. And computing d requires knowing
φ(n), which requires knowing p and q, which requires factoring n.

The modular exponentiation (m^e mod n) is fast to compute but the reverse is
hard without the private key — that asymmetry is the whole foundation of RSA.
