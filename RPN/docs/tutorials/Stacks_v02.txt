# Stacks Tutorial

The stack is the heart of an RPN calculator.

Values are pushed onto the stack, and commands operate on the values near the top.

Example:

    RPN> 3
    RPN> 4
    RPN> +

Result:

    7

The + command removes the top two values, adds them, and pushes the result.

---

## Stack Order

If you enter:

    3
    4

the stack contains:

    1: 4
    2: 3

The most recent value is on top.

Think of the stack like a pile of face-up cards.

---

## Basic Stack Commands

Common stack commands include:

    dup      duplicate the top value
    drop     remove the top value
    swap     exchange the top two values
    clear    clear the stack

Example:

    RPN> 5
    RPN> dup
    RPN> *

Result:

    25

This computes 5 squared.

---

## Why Stack Commands Matter

Stack commands let you rearrange values without retyping them.

Example:

    RPN> 10
    RPN> 3
    RPN> swap
    RPN> -

Without swap:

    10 3 -

means:

    10 - 3

With swap:

    10 3 swap -

means:

    3 - 10

Stack order matters.

---

## Multiple Stacks

This calculator supports multiple named stacks.

That means you can keep separate work areas.

For example, you might have:

    default
    scratch
    primes
    financial

The default stack is enough for normal use, but named stacks are useful for experiments.

---

## Stack Management

Use:

    RPN> help category stack

to see the stack-related commands available in your version.

Common stack-management ideas include:

    create a new stack
    switch stacks
    list stacks
    delete a stack
    save stack contents
    restore stack contents

    RPN> stack
        Report the current stack.

    RPN> stack <name>
        Switch to (or create) a named stack.

    RPN> stack list
        List all named stacks and their depths.

---

## Common Mistakes

### Not Enough Values

If you enter:

    RPN> 3
    RPN> +

the calculator will complain because + needs two values.

The stack is left unchanged.

### Wrong Order

Subtraction and division depend on order.

    10 2 -

is different from:

    2 10 -

When results look strange, check stack order first.
