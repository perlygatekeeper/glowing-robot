
## Guiding Principles

These principles guide the long-term evolution of the RPN Calculator.

* The command registry is the authoritative source for command metadata.
* Documentation should be generated whenever practical rather than
  duplicated.
* Preserve backward compatibility whenever practical.
* Favor simple user-facing semantics over clever implementation.
* Executable text and executable structure are distinct concepts.
* Token Strings are convenience.
* Code Blocks are structure.
* Introduce new abstractions before introducing commands that depend on them.

## Version 3.8.8 — CodeBlock Object Foundation

### Theme

Introduce CodeBlock as a first-class executable value object.

This release establishes the object model and persistence foundation for
future execution features. CodeBlocks do not need to replace functions
immediately. Instead, they become a structured representation that later
features can consume.

### Goal

  Introduce CodeBlocks as first-class executable value objects while
  preserving existing Function (Token String) behavior.

### Design Philosophy/Principle

  Token Strings are convenience.
  Code Blocks are structure.
  Functions remain lightweight executable Token Strings.
  CodeBlocks become structured executable values.

### Goals

[x] Add RPN::CodeBlock value object.
[x] Support anonymous CodeBlocks.
[x] Support named CodeBlocks.
[x] Implement save/load support.

### Non-Goals

[0] Do not replace existing functions.
[0] Do not overload def yet.
[0] Do not implement execution semantics.
[0] Do not require all CodeBlocks to be named.
[0] Do not implement looping.
[0] Do not implement map/filter/reduce yet.
[0] Do not introduce blessed Number or String objects.
[0] Do not solve full execution semantics yet unless naturally required.

### Future Use

CodeBlocks will provide the foundation for:

        call
        named executable values
        map
        filter
        reduce
        branching
        recursion
        structured execution
        future debugging/introspection

### Notes

RPN::Functions currently represents named Token Strings.
RPN::CodeBlock will represent structured executable values.
RPN::CodeBlocks will represent named CodeBlock storage.

These concepts are related, but intentionally distinct.

## Purpose

This release establishes the object model only.

It intentionally postpones execution semantics until the following
release.

## Deferred

[ ] foreach

    Deferred deliberately. The read-only iteration semantics require
    more design work before becoming part of the language.

[ ] loop / break / continue

    Reserved for a future release. These require a loop-control model

## Purpose

Version 3.8.9 is the release where CodeBlocks become useful as
executable values and where the calculator gains the foundation for
higher-order stack programming.
