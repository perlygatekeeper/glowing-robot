RPN Calculator
Release Notes
Version 3.1.0
=============

## Release Date

June 2026

## Overview

Version 3.1.0 introduces a dedicated constants subsystem and user-defined
constants. Constants are now managed independently of the core calculator,
allowing users to create, save, load, and organize their own mathematical,
scientific, and application-specific values.

## Highlights

* New RPN::Constants module
* User-defined constants
* Persistent custom constants
* Load and save constant files
* Protection of built-in constants
* Example constant packs
* Expanded automated test coverage

## New Architecture

New module:

lib/RPN/Constants.pm

Responsibilities:

* Built-in constants
* User constants
* Constant lookup
* Constant creation
* Constant deletion
* Loading constant files
* Saving constant files
* Protection of built-in values

## Built-In Constants

The following mathematical constants are available by default:

pi
e
phi

r2
r3
r5
r7

Where:

phi = Golden Ratio
r2  = sqrt(2)
r3  = sqrt(3)
r5  = sqrt(5)
r7  = sqrt(7)

## Custom Constants

Users may create their own constants:

const gravity 9.80665
const answer 42

and use them directly:

gravity
answer

Custom constants are stored separately from built-in constants and may be
saved and reloaded across sessions.

## Constant Management

New commands:

const
    List all constants

const <name>
    Display or push a constant

const <name> <value>
    Define a user constant

delconst <name>
    Delete a user constant

loadconst <file>
    Load constants from a file

saveconst
    Save user constants

## Built-In Constant Protection

Built-in constants cannot be modified or removed.

Examples:

const pi 3

delconst pi

will generate warnings and leave the built-in constant unchanged.

## Constant Files

Constant files support:

* Blank lines
* Full-line comments
* Trailing comments

Example:

# Physics Constants

c = 299792458          # m/s
g = 9.80665            # m/s^2
R = 8.31446261815324   # J/(mol*K)

This format allows constant files to serve as both data and documentation.

## Example Constant Packs

The project now includes example constant collections:

constants/mathematics.txt
constants/physics.txt
constants/chemistry.txt

These files demonstrate recommended formatting and provide useful
starting points for customization.

## Persistence Improvements

Constant persistence has been integrated into the existing persistence
framework.

The calculator now supports:

* History persistence
* Stack persistence
* Constant persistence

using separate storage files.

## Testing

New automated test coverage includes:

Custom constant creation
Constant lookup
Constant deletion
Built-in protection
Constant file loading
Constant file saving
Persistence validation

All tests pass successfully.

## Upgrade Notes

No compatibility-breaking changes were introduced.

Existing 3.0.0 functionality remains fully supported.
