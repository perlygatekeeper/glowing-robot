RPN Calculator
Release Notes
Version 3.5.0
=============

## Release Date

June 2026

## Overview

Version 3.5.0 introduces user-defined functions.

Functions allow users to define, save, load, and execute custom
RPN command sequences using simple names. Combined with variables,
constants, persistence, and file I/O, this release significantly
expands the calculator from a collection of commands into a
programmable RPN environment.

## New Features

User-Defined Functions

```
def
undef

functions

savefuncs
loadfuncs
```

## Function Definition

Functions are created using:

```
def <name> <body>
```

Examples:

```
def sq duplicate multiply

def cube duplicate duplicate multiply multiply

def addtwo 2 add
```

## Function Execution

Functions execute simply by entering their name:

```
3
sq
```

Result:

```
9
```

Functions may use any existing calculator command and may be
combined with variables, constants, and user-defined constants.

## Function Persistence

Functions are automatically loaded at startup and saved at exit.

Manual control is also available:

```
savefuncs
loadfuncs
```

Functions are stored in:

```
~/.rpn_functions
```

or the file specified by:

```
RPN_FUNCTIONS
```

## Function Management

List all functions:

```
functions
```

Delete a function:

```
undef <name>
```

## Namespace Protection

The calculator now enforces a unified namespace policy.

The following names are reserved:

```
Commands
Command aliases
Constants
Variables
Functions
```

Functions cannot be defined using names already occupied by
commands, aliases, constants, or variables.

Variables cannot be created using names already occupied by
commands or aliases.

Examples:

```
def add ...
def pi ...
store sqrt
```

All of the above are rejected.

## Lookup Order

Input resolution now follows the following order:

```
1. Exact command names
2. Exact command aliases
3. User-defined functions
4. Constants
5. Variables
6. Command abbreviations
7. Unknown input
```

This prevents abbreviation collisions while preserving the
convenience of abbreviated commands.

For example:

```
pi
```

always resolves to the mathematical constant:

```
3.141592653589793
```

rather than the abbreviation for:

```
pick
```
